Problem & Goals

Problem

Traditional STEAM content is often visually biased. TocaCiencia focuses on improving accessibility for visually impaired students through tactile exploration combined with interactive digital content triggered by NFC.

Goals

  • Create a tactile-to-digital learning flow using NFC triggers.
  • Support curriculum-aligned content with scalable configuration.
  • Ensure a robust and simple setup for educators and students.

System Overview

Key Features

  • Tactile 3D printed models for macro–micro exploration.
  • NFC-based identification to trigger content in the app.
  • Accessible UI and guided learning interactions.

Pipeline / Flow

  • Scan NFC tag → identify model/content
  • Load lesson module → present guided interaction
  • Track progress / provide feedback

Technical Implementation

Architecture

  • Data-driven content mapping (NFC ID → lesson module).
  • Decoupled NFC integration from presentation layer.
  • Reusable lesson components for scalability.

Reliability & Performance

  • Error handling for missing/invalid NFC reads.
  • Clear fallback UX when content is unavailable.
  • Simple configuration for educators (low setup friction).
Highlight: The NFC-to-content mapping was designed to be extensible, enabling new models and lessons to be added without changing the app’s core logic.
