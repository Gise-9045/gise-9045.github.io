Project Overview

Context

Beach Cleaning Simulator is a simulation prototype where the player cleans a beach using a vacuum tool, then sorts collected waste into appropriate bins. It focuses on clear feedback, satisfying interactions, and progression tasks.

Goals

  • Create a satisfying collection loop with readable feedback.
  • Build an interaction system that scales across objects.
  • Track tasks/progression to support structured gameplay.

Gameplay & Design Focus

Core Mechanics

  • Vacuum-based object collection.
  • Waste classification and sorting.
  • Task-driven objectives (clean zones / quotas).

Design Pillars

  • Clarity: players always know what they collected.
  • Feedback: strong response to actions (SFX/VFX/UI).
  • Progression: goals push exploration and completion.

Technical Implementation

Architecture

  • Reusable interaction components (trace + interface).
  • Object states (collectable / collected / sorted).
  • Data-driven categorization for waste types.

Key Systems

  • Vacuum tool logic (range, suction, target validation).
  • Inventory / collected items tracking.
  • Objective system (tasks, quotas, zone completion).
Highlight: The interaction framework was designed to be extendable, allowing new collectible types and sorting rules without rewriting the core vacuum tool.
